# AI 활용 내역

본 문서는 프로젝트 수행 과정에서 AI를 어떻게 활용했는지에 대한 상세한 기록과 회고입니다.

---

## 1. 사용한 AI 도구와 사용 이유

**주요 도구: Cursor IDE **

단순한 챗봇 형태의 도구나 CLI 기반의 도구(예: Claude Code) 대신 **Cursor IDE**를 메인으로 선택했습니다. 그 이유는 다음과 같습니다.

- **맥락(Context) 주입의 용이성**: 특정 코드나 파일을 물어볼 때 파일 경로와 줄 번호를 일일이 복사해서 텍스트로 입력할 필요 없이, 에디터 상에서 `@파일이름` 태그나 코드 드래그만으로 즉시 정확한 컨텍스트를 AI에게 전달할 수 있습니다.
- **통합된 워크플로우**: 에러 원인 파악, 코드 생성, diff 확인, 그리고 실제 파일에 수정 사항을 적용하는 모든 과정이 하나의 GUI 창 안에서 물 흐르듯 이어지기 때문에, 단기간에 빠른 생산성을 내야 하는 과제 환경에서 효율적이라 판단하여 Cursor를 선택하였습니다.

---

## 2. 프롬프트, 의사결정 과정, 컨텍스트 파일 활용

AI가 엉뚱한 코드를 생성하는 것을 막고, 프로젝트의 방향성에 맞는 결과를 도출하기 위해 **명확한 컨텍스트 제공과 단계적 프롬프팅**을 핵심 전략으로 삼았습니다.

- **컨텍스트 파일 기반 프롬프팅**: 무작정 "차트 만들어줘"라고 요청하지 않았습니다. 항상 `@ARCHITECTURE.md`, `@REQUIREMENTS.md`, `@WORKFLOW.md` 문서를 멘션하여, 프로젝트의 설계 원칙과 세부 요구사항을 AI가 먼저 숙지한 상태에서 코드를 작성하도록 유도했습니다.
  - _예시 프롬프트_: _"@ARCHITECTURE.md @REQUIREMENTS.md 참고해서 @WORKFLOW.md:200-209 플랫폼별 성과 차트 작업해줘"_

- **Rules 세팅 **: 대규모 언어 모델(LLM)들은 과거의 방대한 데이터로 학습되어 있어 종종 구버전의 React 패턴이나 예전 방식의 TanStack Query 코드를 작성하는 경향이 있습니다. 이를 방지하고 프로젝트 전반에 걸쳐 일관된 코드 품질을 유지하기 위해 `.cursor/rules`에 `react-best-practices` 및 `tanstack-query-best-practices` 룰을 명시적으로 적용했습니다. 이를 통해 AI가 최신 버전의 React 컴포넌트 렌더링 최적화, 커스텀 훅 설계, 서버 상태 관리 등에 대한 Best Practice를 AI가 따르도록 하였습니다.

---

## 3. AI 결과물을 직접 수정한 판단 사례

AI가 생성한 코드가 비록 정상 작동하더라도, **아키텍처 관점에서 유지보수성이 떨어지거나 과도하게 최적화된 경우 즉각적으로 개입하여 방향을 수정(Refactoring)**했습니다. 특히 무분별한 최적화보다는 최적화 비용(메모리 할당 및 의존성 비교 연산)을 고려하여 불필요한 메모이제이션(`useMemo`, `useCallback`)을 걷어내고, 참조 무결성이나 실제 병목 해소가 필요한 시점에만 전략적으로 최적화를 적용하도록 교정했습니다.

---

## 4. 나의 AI 활용 역량

저는 AI를 단순히 코드 작성 도구로 사용하는 것을 넘어, **자체적인 AI 에이전트 스킬(Agent Skill)을 개발하고 고도화하여 워크플로우를 자동화하는 역량**을 갖추고 있습니다.

대표적으로 프로젝트 내에 **"PR Creator" 에이전트 스킬**을 자체 구축하여 운용했습니다. 에이전트에게 "PR 만들어줘"라고 요청하면, 별도의 외부 패키지 설치 없이 OS 기본 쉘(macOS/Linux의 `sh` 및 Windows의 `PowerShell`)만을 활용하여 `파일 분류 → 서브에이전트 병렬 diff 요약 → PR 초안 생성 → GitHub CLI PR 생성`의 전 과정을 자동으로 수행합니다.

이 에이전트 스킬을 구축하며 다음과 같이 3단계(Phase)에 걸쳐 AI의 추론 능력을 고도화했습니다:

1. **Phase 1 [결정론적 처리 분리]**
   - 프로젝트 스캔 및 파일 파싱과 같이 명확한 규칙이 있는 로직은 외부 Bash/PowerShell 스크립트로 위임했습니다.
   - 이를 통해 에이전트의 추론 범위를 '코드 요약 및 맥락 파악'이라는 핵심 태스크로 좁혀, 할루시네이션 발생률을 대폭 감소시키고 토큰 비용을 절감했습니다.
2. **Phase 2 [관심사 분리]**
   - 비대해진 프롬프트 지침을 `Workflow(SKILL.md)`, `Domain Knowledge(references/)`, `Assets(JSON/Data)` 구조로 역할을 나누어 설계했습니다.
   - 프롬프트 엔지니어링의 구조화를 통해 에이전트 스킬의 유지보수성과 확장성을 크게 향상시켰습니다.
3. **Phase 3 [Evals 도입]**
   - AI의 결과물에 대한 주관적 판단을 배제하기 위해 정량적인 품질 검증 체계를 구축했습니다.
   - 주요 케이스별로 4개의 시나리오와 기대 출력 조건을 명시하고, 'AI-as-judge' 패턴을 활용해 결과물의 누락 섹션이나 논리적 오류를 자동으로 검증하는 프로세스를 구현했습니다.
