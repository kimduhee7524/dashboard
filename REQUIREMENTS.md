> 아키텍처/폴더 규칙은 [ARCHITECTURE.md](ARCHITECTURE.md), 구현 일정은 [WORKFLOW.md](WORKFLOW.md) 참조.

## 1. 데이터 스키마

### 1.1 Domain 타입 (정규화 후 Target)

normalize 완료 후 모든 코드에서 사용하는 타입.

```ts
interface Campaign {
  id: string;
  name: string;
  platform: 'Google' | 'Meta' | 'Naver';
  status: 'active' | 'paused' | 'ended';
  budget: number;
  startDate: string; // YYYY-MM-DD
  endDate: string | null; // YYYY-MM-DD
}

interface DailyStat {
  id: string;
  campaignId: string;
  date: string; // YYYY-MM-DD
  impressions: number;
  clicks: number;
  conversions: number;
  cost: number;
  conversionsValue: number | null;
}
```

### 1.2 Raw 데이터 특성 (db.json)

`db.json`의 원본 데이터에는 아래와 같은 예외가 포함되어 있다. **normalize에서 일괄 처리**해야 한다.

- platform: `"네이버"`, `"facebook"`, `"Facebook"` 등 비표준 표기 혼재
- status: `"stopped"`, `"running"` 등 비표준 표기 혼재
- budget: 문자열(`"2000000원"`)이거나 null인 경우 존재
- startDate: 구분자가 `/`인 경우 존재 (`2026/01/15`)
- daily_stats: 동일 `(campaignId, date)` 중복 행 존재
- 수치 필드(impressions, clicks, cost 등): null인 경우 존재
- conversionsValue: null 허용 (Domain 타입에서도 null 유지)

## 2. 파생 지표 공식

필터 조건에 해당하는 `dailyStats`를 **먼저 합산**한 뒤, 합산 결과로 지표를 계산한다.

| 지표         | 공식                                | 분모 0일 때 |
| ------------ | ----------------------------------- | ----------- |
| **CTR (%)**  | `(총 클릭수 / 총 노출수) * 100`     | `0`         |
| **CPC (원)** | `총 집행비용 / 총 클릭수`           | `0`         |
| **ROAS (%)** | `(총 전환가치 / 총 집행비용) * 100` | `0`         |

- `safeDivide(numerator, denominator)` 유틸 사용 (분모 0이면 0 반환)
- 신규 캠페인(daily_stats 없음): 지표는 `0` 또는 `-`으로 표시

## 3. 글로벌 필터 (필수)

### 3.1 필터 항목

| 필터      | 초기값                     | 동작                                                                                               |
| --------- | -------------------------- | -------------------------------------------------------------------------------------------------- |
| 집행 기간 | **당월 1일 ~ 말일**        | 캠페인의 `startDate~endDate`가 선택 기간과 **겹치는** 캠페인만 포함. 해당 기간의 dailyStats만 집계 |
| 상태      | 전체 (빈 배열 = 필터 없음) | 다중 선택                                                                                          |
| 매체      | 전체 (빈 배열 = 필터 없음) | 다중 선택                                                                                          |
| 초기화    | -                          | 모든 조건을 초기값으로 복구                                                                        |

### 3.2 필터 규칙

- 필터 조건은 **AND** 조합으로 적용
- 필터 변경 시 하단의 **모든 차트와 테이블이 실시간 동기화**
- 빈 배열(`[]`)은 "전체 선택"으로 해석 (해당 필터 미적용)

### 3.3 날짜 필터 판정 로직

캠페인의 집행 기간(`startDate ~ endDate`)이 선택 기간과 **겹치면** 포함:

```
campaignStart <= filterEnd AND (campaignEnd >= filterStart OR campaignEnd === null)
```

- `endDate`가 null인 캠페인은 "아직 종료되지 않은 캠페인"으로 간주하여 포함

## 4. 일별 추이 차트 (필수)

- X축: 날짜, Y축: 수치, 범례(Legend) 필수
- **메트릭 토글**: 노출수 / 클릭수
  - 초기값: **둘 다 활성화** (중복 선택 가능)
  - 제약: **최소 1개는 반드시 선택 유지**
- 호버 시 해당 날짜의 수치를 **툴팁**으로 표시
- 차트 유형은 자유 (Recharts LineChart 권장)

## 5. 캠페인 관리 테이블 (필수)

### 5.1 테이블 컬럼

| 컬럼                     | 정렬 가능 |
| ------------------------ | --------- |
| 캠페인명                 | X         |
| 상태                     | X         |
| 매체                     | X         |
| 집행기간 (시작일~종료일) | O         |
| 총 집행금액              | O         |
| CTR (%)                  | O         |
| CPC (원)                 | O         |
| ROAS (%)                 | O         |

### 5.2 기능

- **정렬**: 오름차순 / 내림차순 토글
- **검색**: 캠페인명 실시간 검색
  - **테이블에만 적용** (차트에는 영향 없음)
  - 검색어 상태는 **로컬 상태**(컴포넌트/훅 내부)로 관리 (글로벌 store에 넣지 않음)
  - 검색 결과 건수 / 전체 건수 표시
- **페이지네이션**: 1페이지당 **10건**
- **일괄 상태 변경** (평가 포인트)
  - 체크박스로 캠페인 선택
  - 드롭다운으로 상태(진행중/일시중지/종료) 일괄 변경
  - PATCH API → `invalidateQueries`로 즉시 반영

## 6. 캠페인 등록 모달 (필수)

### 6.1 인터랙션

- [캠페인 등록] 버튼 클릭 시 모달 노출
- 등록 성공 시 **별도의 새로고침 없이** 목록과 차트에 즉시 반영 (`invalidateQueries`)

### 6.2 자동 설정값

- status: `active`(진행 중)로 고정
- id: 고유 식별값 자동 생성

### 6.3 폼 필드 및 검증 규칙 (Zod)

| 필드      | 타입   | 필수 | 검증 규칙                              |
| --------- | ------ | ---- | -------------------------------------- |
| 캠페인명  | 텍스트 | O    | 2자 ~ 100자                            |
| 광고 매체 | 선택   | O    | `Google` / `Meta` / `Naver` 중 택 1    |
| 예산      | 숫자   | O    | 정수, 100원 ~ 10억원                   |
| 집행 금액 | 숫자   | O    | 정수, 0원 ~ 10억원, **예산 초과 불가** |
| 시작일    | 날짜   | O    | -                                      |
| 종료일    | 날짜   | O    | **시작일 이후**                        |

- **교차 검증**: 집행 금액 <= 예산 (Zod `.refine()` 사용)
- **교차 검증**: 종료일 > 시작일 (Zod `.refine()` 사용)
- 검증 실패 시 해당 필드 하단에 에러 메시지 표시

### 6.4 신규 캠페인의 지표 처리

- 신규 캠페인은 `daily_stats`가 없으므로 테이블에서 지표는 **0 또는 `-`으로 표시** 가능
- `calcMetrics`에 빈 배열이 들어올 때의 fallback 처리 필요

## 7. 플랫폼별 성과 차트 - 도넛 (선택, 가산점)

- **메트릭 토글**: 비용 / 노출수 / 클릭수 / 전환수 (기본값: **비용**, 단일 선택)
- 플랫폼별(Google / Meta / Naver) 비중 표시
- 메트릭 수치와 비중(%) 동시 표기
- **양방향 필터 연동** (평가 포인트)
  - 도넛 영역 클릭 → `store/filter-store.ts`의 `togglePlatform()` 호출
  - 상단 글로벌 필터에 즉시 반영 → 모든 위젯 갱신
  - 이미 선택된 플랫폼 재클릭 시 필터 해제

---

## 8. 캠페인 랭킹 Top3 - 바 (선택, 가산점)

- **메트릭 토글**: ROAS / CTR / CPC (기본값: **ROAS**, 단일 선택)
- 선택한 메트릭 기준 상위 3개 캠페인 노출
- **정렬 방향 주의**
  - ROAS / CTR: **높을수록** 상위 랭킹
  - CPC: **낮을수록** 상위 랭킹

## 9. API 엔드포인트 (MSW)

| Method | Endpoint                | 용도                |
| ------ | ----------------------- | ------------------- |
| GET    | `/campaigns`            | 캠페인 전체 조회    |
| GET    | `/daily_stats`          | 일별 성과 전체 조회 |
| POST   | `/campaigns`            | 캠페인 등록         |
| PATCH  | `/campaigns/bulk/status`| 캠페인 다중 상태 변경|
| PATCH  | `/campaigns/:id/status` | 캠페인 상태 변경    |

- `db.json` 원본 수정 금지
- import 방식 금지 — 반드시 **비동기 API 통신**(MSW)
- 상태 영속성: 브라우저 메모리 내 유지 (새로고침 시 초기화 허용)
